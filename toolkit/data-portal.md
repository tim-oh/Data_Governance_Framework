# Data portal

**Pillar:** Access | **Building block:** Open data | **Priority:** Intermediate

## What it is

The premise of open data is easy, universal access. An online portal is a sharing modality that can provide such access. The US Office of Management and Budget (2013) provides specific guidance on open data and related sharing policies.

## Components

| Component | Description |
|-----------|-------------|
| Simple website for data downloads | Open data does not usually require user authentication. A simple website that provides for data downloads, for items that have passed the release approval process, is the core requirement. |
| Existing websites or portals | Instead of creating a site from scratch, which can be relatively costly, cumbersome and require scarce ICT resources, the data can be made available through existing websites or portals. |
| User accounts | If the data access policy permits provision of custom data sets (e.g. micro-data access) for requesting parties that have successfully completed a data access request, a website with user accounts through which the permissioned dataset can be downloaded can be a useful implementation aspect. |
