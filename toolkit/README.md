# Toolkit

The Toolkit provides a library of implementation mechanisms — the practical tools, templates, checklists, and standard operating procedures that operationalise the building blocks across the four pillars.

Each toolkit page covers one mechanism and follows a consistent structure: what it is, how to use it, inputs and outputs, priority, and links back to the relevant pillar and building block.

## How to use the toolkit

- **Browsing by pillar:** see the [SUMMARY](SUMMARY.md) — mechanisms are grouped by pillar and building block
- **Finding a specific mechanism:** use the search function or navigate from the relevant building block page in the [Pillars space](../pillars/README.md)
- **Priority tags:** each mechanism is tagged Immediate or Intermediate — start with Immediate priorities

## Page template

Each toolkit page follows this structure:

1. **What it is** — brief description of the mechanism
2. **How to use it** — step-by-step guidance
3. **Inputs and outputs** — what you need to have in place, and what the mechanism produces
4. **Priority** — Immediate or Intermediate
5. **RACI** — roles and responsibilities for this mechanism
6. **Related building blocks** — links back to the pillar reference
7. **Template/checklist** — downloadable or inline template where applicable
