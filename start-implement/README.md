# Start here

This space is the entry point for practitioners implementing the Data Governance Framework. It provides a concise overview of the framework and a practical five-step implementation cycle to guide how you apply it in your context.

## Where to begin

- New to the framework? Start with [Framework at a glance](framework-at-a-glance.md) for a one-page orientation.
- Ready to implement? Follow the [five-step implementation cycle](#the-five-step-implementation-cycle) below.
- Looking for reference material on specific pillars or building blocks? Go to the [Pillars space](../pillars/README.md).
- Looking for templates and checklists? Go to the [Toolkit space](../toolkit/README.md).

## The five-step implementation cycle

The framework is designed to be customised for each national context through a structured five-step process:

| Step | Title | Focus |
|------|-------|-------|
| [1](step-1-convene-stakeholders.md) | Convene stakeholders | Governance and accountability |
| [2](step-2-define-scope.md) | Define scope | Policy and contextual fit |
| [3](step-3-profile-organisations.md) | Profile organisations | Current status and gaps |
| [4](step-4-plan-implementation.md) | Plan implementation | Prioritisation and project design |
| [5](step-5-implement.md) | Implement the plan | Project management and iteration |
