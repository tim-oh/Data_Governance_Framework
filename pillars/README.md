# Pillars of Data Governance

This space provides the reference documentation for the four pillars of the Data Governance Framework. Each pillar is broken down into building blocks, which are in turn operationalised through specific implementation mechanisms.

## The four pillars

| Pillar | What it covers |
|--------|----------------|
| [1. Management](management/README.md) | Policies and procedures for collecting, processing, storing, and deleting data throughout its lifecycle |
| [2. Quality](quality/README.md) | Procedures and processes that ensure data is accurate, consistent, complete, and reliable |
| [3. Access](access/README.md) | Policies and mechanisms that ensure authorised users can retrieve and use data while maintaining privacy and compliance |
| [4. Security](security/README.md) | Measures and protocols to protect data from unauthorised disclosure, loss, alteration, and other threats |

## Framework levels

Each pillar is structured across five levels of detail:

- **Level 1 — Pillars:** the four broad themes (this space)
- **Level 2 — Building blocks:** concrete sub-areas within each pillar
- **Level 3 — Implementation mechanisms:** specific tools, policies, and processes (see [Toolkit](../toolkit/README.md))
- **Level 4 — Roles and responsibilities:** RACI assignments per mechanism (see [Roles](roles-and-responsibilities.md))
- **Level 5 — Tasks:** specific actions by actor
