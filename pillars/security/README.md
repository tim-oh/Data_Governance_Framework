# Pillar 4: Security

Data security covers the measures and protocols implemented to protect data from unauthorised disclosure, access, accidental or unlawful destruction, loss, alteration, and other threats. This pillar draws on the ISO/IEC 27000 family of information technology security standards (ISO & IEC, 2022).

Security is structured around four types of measures: technical, organisational, physical, and people. Any technical process used to collect, process, store, use, or share data should employ robust security mechanisms.

## Building blocks

| Building block | What it covers |
|----------------|----------------|
| [Technical measures](technical.md) | Encryption, authentication, network security, and other technical controls |
| [Organisational measures](organisational.md) | Incident response, supplier risk management, and regular security reviews |
| [Physical measures](physical.md) | Physical access controls, device protection, and backup and recovery |
| [People measures](people.md) | Staff training and insider threat management |

## Roles overview

**IT services (IT)** carry primary responsibility for most security mechanisms. The **governance council (GC)** is accountable. **Data administrators (DA)**, **programme administrators (PA)**, **data collectors (DC)**, and **data providers (DP)** are responsible for implementing security measures within their own operations.
